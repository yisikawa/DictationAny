# DictationAny 企画案

## 概要

DictationAny は、英語を中心とした任意の文書を表示・読み上げし、ユーザーが聞き取った内容を入力して、原文との差分を比較・評価するディクテーション学習アプリです。

単なる正誤判定ではなく、聞き落とし、スペルミス、語順違い、句読点違いなどを可視化し、ユーザーが自分の弱点を把握しながら反復練習できることを目指します。

## コンセプト

ユーザーが文章を「聞く、入力する、比較する、復習する」という学習ループを気持ちよく回せるアプリにします。

英語学習だけに限定せず、将来的には多言語のリスニング、語学教材、教師向け課題作成にも対応できる設計を想定します。

## 想定ユーザー

- 英語のリスニング力を鍛えたい学習者
- TOEIC、IELTS、英検などの試験対策をしたいユーザー
- ニュース記事、例文、教材文を使って練習したいユーザー
- 英語以外の言語でディクテーション練習したい多言語学習者
- 教材文を登録して生徒に課題を出したい教師

## 主な利用シーン

- サンプル英文を使って毎日 5 分だけ聞き取り練習する
- 自分で登録した英文記事を TTS で読み上げ、全文ディクテーションする
- 間違えた文だけを復習し、苦手な単語や表現を重点的に練習する
- 教師が教材を作成し、生徒に課題として配布する

## 基本機能

### 教材入力

- ユーザーによる文章の直接入力
- テキストファイルからの取り込み
- レベル別、ジャンル別のサンプル教材
- 将来的な PDF 取り込み
- 将来的な Web 記事 URL からの本文抽出
- 将来的な YouTube 字幕からの教材作成
- 将来的な Udemy 講座の字幕・文字起こしからの教材作成

### 外部教材取り込み

YouTube や Udemy などの動画教材から、字幕や文字起こしを取り込み、ディクテーション教材として利用できるようにします。

取り込み対象:

- 動画タイトル
- 動画 URL
- チャプターまたはセクション名
- 字幕テキスト
- 字幕のタイムスタンプ
- 言語

YouTube では、公開動画の字幕またはユーザーが利用権限を持つ字幕データを教材化します。Udemy では、ユーザーが受講権限を持つ講座の字幕・文字起こしを、各サービスの規約に従って取り込みます。

初期段階では、字幕ファイルのアップロードや貼り付けによる取り込みを優先し、外部サービスとの直接連携は後続機能として扱います。

### 音声再生

- Text-to-Speech による文章読み上げ
- 再生速度変更
  - 0.5x
  - 0.75x
  - 1.0x
  - 1.25x
- 文単位、段落単位での再生
- リピート再生
- 一時停止、巻き戻し、数秒戻る操作

### ディクテーション入力

- 聞こえた内容を入力するテキストエリア
- 全文入力モード
- 文ごと入力モード
- 穴埋めモード
- ヒント表示の有無を切り替え

### 比較・採点

- 原文とユーザー入力文を比較
- 正解、一部一致、誤字、抜け、余分な単語を色分け
- 単語一致率の表示
- 文字一致率の表示
- 聞き落とし単語数の表示
- 大文字小文字や句読点を評価対象に含めるか選択可能

### フィードバック

- 間違えた単語一覧
- 抜けやすい冠詞、前置詞などの傾向表示
- 類似音で間違えた可能性のある語の表示
- 原文と入力文の並列表示
- 差分ハイライト

### 復習

- 間違えた文だけ再挑戦
- 苦手単語リスト
- 過去のスコア履歴
- 完了済み教材の履歴
- 学習時間や連続学習日数の記録

## 学習モード

### Standard Mode

音声を聞いて全文を入力する基本モードです。

### Sentence Mode

1 文ずつ聞いて入力する初心者向けモードです。

### Blind Mode

原文を表示せず、音声だけを聞いて入力するモードです。

### Shadowing Mode

原文を見ながら音声に合わせて発声練習するモードです。

### Fill-in-the-Blank Mode

一部の単語を空欄にして入力するモードです。

### Exam Mode

再生回数や制限時間を設け、採点結果のみを表示する試験形式のモードです。

## 評価ロジック案

初期版では、完全一致だけに頼らず、単語単位 diff と Levenshtein 距離を組み合わせて評価します。

評価対象の例:

- 完全一致
- スペル違い
- 単語抜け
- 余分な単語
- 語順違い
- 句読点違い
- 大文字小文字違い

将来的には AI 評価を導入し、意味は近いが原文とは異なる入力や、聞き間違えの原因になりやすい音の傾向も説明できるようにします。

## MVP

最初の実用版では、以下に絞って実装します。

1. 文章入力
2. TTS 読み上げ
3. ユーザー入力
4. 原文との差分表示
5. スコア表示
6. 練習履歴の保存

この範囲で「聞く、書く、間違いを見る、もう一度聞く」というコア体験を成立させます。

## プロトタイプ目標

最初のプロトタイプでは、1 つの英文を入力し、読み上げ、ユーザーが聞き取って入力し、差分とスコアを見るところまでを実装します。

アプリの価値は派手な機能ではなく、学習ループの滑らかさにあります。まずは短い英文で素早く練習し、すぐに結果を確認できる体験を丁寧に作ります。
